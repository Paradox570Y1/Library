## **1. Cloud Computing Basics**

### 🔹 1.1 History of Cloud Computing

#### 🧠 Key Points:

- **1960s** – Idea of **utility computing** (John McCarthy): computing as a public utility.
    
- **1999** – **Salesforce.com** launched SaaS model.
    
- **2006** – **Amazon Web Services (AWS)** introduced EC2 (Elastic Compute Cloud), marking the commercial boom of cloud.
    
- **2008-Present** – Google Cloud, Microsoft Azure, IBM Cloud, etc. follow with advanced services.
    

#### 📌 Analogy:

Think of electricity – you don't generate your own power; you use a utility company. Similarly, cloud lets you use computing power on-demand.

---

### 🔹 1.2 Characteristics of Cloud Computing

| Characteristic             | Meaning                                                        |
| -------------------------- | -------------------------------------------------------------- |
| **On-demand self-service** | Users can provision resources without human interaction.       |
| **Broad network access**   | Services accessible via internet across various devices.       |
| **Resource pooling**       | Multi-tenant model; resources shared across users dynamically. |
| **Rapid elasticity**       | Ability to scale up/down quickly as per need.                  |
| **Measured service**       | Usage is monitored and billed (pay-as-you-go model).           |

🧠 **Example:** Netflix leverages AWS to stream millions of videos, scaling automatically during peak hours.

---

### 🔹 1.3 Need for Cloud Computing

- Reduce **capital expenditure** on hardware.
    
- Ensure **business continuity** with backup and disaster recovery.
    
- Enable **global collaboration** (e.g., Google Docs).
    
- Support **agile development** and **faster deployments**.
    

---

### 🔹 1.4 Advantages and Disadvantages

#### ✅ Advantages:

- Cost-effective
    
- Scalable and flexible
    
- Automated updates
    
- Remote access from anywhere
    
- Environmentally friendly (shared resources)
    

#### ⚠️ Possible Disadvantages:

- Internet dependency
    
- Security concerns (data breaches, misconfiguration)
    
- Limited control over infrastructure
    
- Compliance and legal issues for sensitive industries

## **2. Cloud Computing Architecture**

---

### 🔹 2.1 Overview of Cloud Architecture

Cloud architecture refers to the **components** and **subsystems** involved in delivering cloud computing services. It's typically divided into:

#### 🧱 **Two Major Parts:**

1. **Front-End (Client Side)**
    
    - What users interact with (e.g., web browser, mobile app).
        
    - Interfaces with the backend via the internet.
        
2. **Back-End (Cloud Provider Side)**
    - Backend is cloud itself
    - Responsible for storing and processing data.
    - Includes servers, storage systems, databases, and cloud management software.

#### 📊 High-Level Diagram:

```sql
 [User Device]
       |
 [Front-End Interface: Web App / Mobile App]
       |
   === Internet ===
       |
 [Back-End Services]
 ├── Compute (VMs, Containers)
 ├── Storage (Blob, File, DB)
 ├── Network (Virtual networks, Load Balancers)
 └── Cloud Management Tools (Monitoring, Security)

```

---

### 🔹 2.2 Key Components of Cloud Architecture

| Component            | Description                                               | Example                                                 |
| -------------------- | --------------------------------------------------------- | ------------------------------------------------------- |
| **Compute**          | Core processing unit in the cloud; executes applications. | AWS EC2, Google Compute Engine                          |
| **Storage**          | Stores files, data, and backups.                          | Amazon S3(Simple Storage Service), Google Cloud Storage |
| **Database**         | Manages structured/unstructured data.                     | Amazon RDS, Firebase, Cosmos DB                         |
| **Networking**       | Ensures secure and reliable communication.                | Load Balancers, Firewalls, VPNs                         |
| **Virtualization**   | Enables multiple OS on the same physical hardware.        | VMware, Hyper-V, KVM                                    |
| **Management Tools** | Monitor, optimize, and control cloud resources.           | [[CloudWatch]], [[Azure Monitor]]                       |

---

### 🔹 2.3 Cloud Delivery Models

These define **how services are delivered**:

|Model|Description|Examples|
|---|---|---|
|**IaaS (Infrastructure as a Service)**|Provides virtualized infrastructure (VMs, storage, networks).|AWS EC2, Google Compute Engine|
|**PaaS (Platform as a Service)**|Provides a development platform with tools and services.|Google App Engine, Heroku|
|**SaaS (Software as a Service)**|Delivers complete software solutions over the web.|Gmail, Salesforce, Google Docs|

🧠 **Analogy: Pizza-as-a-Service**

|Service Type|You Manage|Cloud Manages|
|---|---|---|
|On-Premises|Everything|Nothing|
|IaaS|Apps, Data|Network, Servers|
|PaaS|Apps|Runtime, OS, Network|
|SaaS|Nothing|Everything|

---

### 🔹 2.4 Deployment Models

| Model               | Description                                              | Use Case                        |
| ------------------- | -------------------------------------------------------- | ------------------------------- |
| **Public Cloud**    | Cloud services offered over the public internet.         | Startups, scalable apps         |
| **Private Cloud**   | Cloud infrastructure operated for a single organization. | Banks, Government               |
| **Hybrid Cloud**    | Combination of public and private clouds.                | Enterprises needing flexibility |
| **Community Cloud** | Shared infrastructure for a specific community.          | Universities, Research groups   |
