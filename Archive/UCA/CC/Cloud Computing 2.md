# What is Cloud?

Cloud computing is a compilation of existing techniques and technologies, packaged within a new infrastructure [[paradigm]] (like old wine in new bottle) that offers improved scalability, elasticity, business agility, faster startup time, reduced management costs, and just-in-time availability of resources.
Cloud offers:
- pay as you go service
- on demand service
- scalable infrastructure


## Types of Cloud?

- Private Cloud 
	- Often refers to internal cloud used within an organization. These clouds offers high security and role based accessed in an organization hierarchy. One example of such cloud is cloud used by Chitkara University.
	- A **private cloud** is a cloud infrastructure dedicated to a single organization. It can be hosted on-premises (inside the organization’s data center) or by a third-party provider, but access is restricted to that organization only.

	- It provides:
		- High security
		- Greater control over data
		- Custom configuration
	**Example:** A university like Chitkara University running its own internal cloud for student records and faculty systems.

- Public Cloud
	- This cloud is designed to handle high user traffic and often used to host public facing resources such as service or website. For example GMAIL services we use is an example of public cloud.
	- A **public cloud** is owned and operated by third-party providers and delivers services over the internet to multiple customers. Resources are shared among users (multi-tenant model).
	- It is ideal for:
		- Hosting websites
		- Applications    
		- Large-scale services
	- High traffic handling
	- Examples:
		- Amazon Web Services (AWS)
		- Microsoft Azure
		- Google Cloud
	 - Services like Gmail run on public cloud infrastructure.


- Hybrid Cloud
	- This cloud is a combination of both private and public cloud. So it gives you the capabilities to host both public facing resources and internal software of the organization.
	- A **hybrid cloud** combines both private and public cloud environments that work together.

	- Organizations use it to:
		- Keep sensitive data in private cloud
		- Host customer-facing apps on public cloud    
		- Scale resources when needed
	- Example: A company stores confidential employee data privately but uses public cloud for its website.

- Community Cloud
	- This cloud is a shared cloud among multiple organization, in which these organizations can use it for providing services as per their workflow to each other internally.
	- A **community cloud** is shared by multiple organizations that have similar requirements (such as security, policy, or compliance needs).
	- It is commonly used by:
		- Government departments
		- Healthcare institutions
		- Financial institutions
	 - The cost and infrastructure are shared among the organizations.


# Types of deployment models

- IaaS
	- IaaS gives you the basic building blocks of IT — like virtual machines, storage, and networks — over the internet. You manage the operating system and applications, but the provider manages the hardware.  
	Think of it like renting an empty house: you get the space, but you set everything up inside.
- SaaS
	- SaaS delivers complete software applications over the internet. You simply use the software through a browser; everything else is handled by the provider.  
	It’s like ordering food at a restaurant — you just consume the service.
- PaaS
	- PaaS provides a ready-made environment where you can develop, run, and manage applications without worrying about servers or infrastructure.  
	It’s like renting a fully equipped kitchen — you just focus on cooking (building your app).
- DaaS
	- DaaS provides a virtual desktop environment that you can access from anywhere using the internet. Your desktop runs on a remote server instead of your personal computer.  
	It’s like carrying your computer with you — but it lives in the cloud.
