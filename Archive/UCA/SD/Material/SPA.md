### 👉 **SPA stands for Single Page Application.**

---

### 🔹 **What is a SPA (Single Page Application)?**

A **Single Page Application** is a web application or website that loads **a single HTML page** and dynamically updates the content **without reloading the entire page**.

Instead of the browser requesting a new page from the server each time you click a link or button, a SPA uses **JavaScript (often with frameworks like React, Angular, or Vue.js)** to update the page content instantly.

---

### 🔹 **How it works (in simple terms):**

1. The browser loads **one HTML page** from the server (usually `index.html`).    
2. All the JavaScript code (and sometimes data) is loaded once.
3. When you navigate inside the app, JavaScript changes the content dynamically without refreshing the whole page.
4. It can **fetch or send data to the server using APIs** (e.g., via `fetch()` or `axios`).

---

### 🔹 **Examples of SPAs:**

- **Gmail** – Clicking on emails doesn't reload the page.
- **Facebook** – Navigating between posts and pages feels instant.
- **Twitter, Instagram (web)** – All interactions happen on the same page.
- **Netflix** – Everything updates inside one page interface.
---

### 🔹 **SPA vs. Traditional Website:**

| Feature             | SPA                               | Traditional Website               |
| ------------------- | --------------------------------- | --------------------------------- |
| Page Reloads        | No (updates dynamically)          | Yes (each click reloads the page) |
| Speed               | Faster after initial load         | Slower due to full reloads        |
| SEO (Search Engine) | Harder (needs setup)              | Easier                            |
| Development Tools   | Needs JS frameworks (e.g., React) | Can work with plain HTML/CSS/JS   |
