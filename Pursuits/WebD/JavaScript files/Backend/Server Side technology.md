# ⭐ **What Is a Server-Side Technology?**

A **server-side technology** is any technology that runs on the **server (backend)** instead of the user’s browser.

When a user sends a request:

### **Client (browser)** → **Server (backend)** → **Database** → **Server** → **Response sent back to client**

A **server-side technology** handles:

- Logic
    
- Processing
    
- Data retrieval
    
- Authentication
    
- Security
    
- Database communication
    
- Creating dynamic content


# ⭐ **What Makes a Technology Server-Side? (Core Characteristics)**

A technology is considered **server-side** if it has these characteristics:

---

## ✅ 1. **Runs on the Server, Not in the Browser**

- Code executes on a backend machine
    
- Client only receives the result (HTML/JSON)
    
- Example: Java Servlet, PHP, Node.js, Python Flask
    

---

## ✅ 2. **Handles Business Logic**

Server decides:

- What to show
    
- What to store
    
- What conditions to check
    
- Which rules to apply
    

Example:  
Checking login username/password → server-side work.

---

## ✅ 3. **Can Connect to Databases**

Only server-side technologies can securely:

- Access databases (SQL, MongoDB)
    
- Run queries
    
- Handle transactions
    

Browsers cannot do this.

---

## ✅ 4. **Manages Security**

Server-side technologies handle:

- Session management
    
- Encryption/decryption
    
- Access control
    
- Validation
    

For example:  
Only server can check if a user is admin.

---

## ✅ 5. **Generates Dynamic Content**

Server can generate:

- Dynamic HTML
    
- JSON responses
    
- API output
    
- Files (PDF, excel)
    

Example:  
JSP dynamically generates HTML after reading data.

---

## ✅ 6. **Processes HTTP Requests**

Server-side tech receives:

- GET
- POST
- PUT
- DELETE
- OPTIONS


Then sends appropriate response.

Example:  
A Servlet’s `doGet()` and `doPost()` handle these.

---

## ✅ 7. **Does Not Depend on User Browser for Execution**

Client-side tech depends on browser (JS, CSS).  
Server-side tech works the same no matter which browser.
