# Software Package registery
A **software package registry** is a massive online database (or repository) that stores reusable pieces of code called "packages," "libraries," or "modules."

**What it's for:** Developers use these registries to easily find, download, and manage code written by other people. Instead of writing everything from scratch (like code to create a PDF or connect to a database), a developer can pull in a pre-built "package" to do the job.

**How it works:** A tool called a **package manager** (like npm, pip, or Maven) connects to the registry to download and install these dependencies.

- **Name:** **npm** (which originally stood for Node Package Manager)
- **Language:** It is the default package registry for **JavaScript** and the Node.js ecosystem.
- **Size:** The npm registry hosts **over 3.1 million** different packages, making it significantly larger than any other.

Simply put:
👉 **NPM allows you to install, update, share, and manage libraries (packages) in your project.**  
👉 It also manages **dependencies** (stuff your project needs to run).
